# 07 - 实战：搭建一个客服机器人

> 综合运用 Coze/Dify，从需求到上线完整走一遍

---

## 场景设定

我们要做一个**电商客服机器人**，能回答：
1. **常见问题**（退货政策、发货时间、支付方式）
2. **订单查询**（查订单状态、物流信息）
3. **商品推荐**（根据需求推荐商品）
4. **转人工**（解决不了时转给真人客服）

---

## 方案一：用 Coze 实现（30 分钟）

### 第一步：准备知识库

准备三个文档：
1. **退货政策**：退换货规则、时间限制、流程
2. **常见问题 FAQ**：发货时间、支付方式、售后
3. **商品目录**：所有商品分类和简介

上传到 Coze 知识库。

### 第二步：编写 Prompt

```你是一个电商客服助手，名叫"小智"。

## 核心规则
1. 先理解用户意图：查订单 / 问政策 / 求推荐 / 要投诉
2. 查知识库获取准确信息
3. 回答要简洁有礼貌
4. 不确定时告诉用户"我帮你转人工"

## 回复格式
- 查订单：订单号 + 状态 + 预计时间
- 问政策：直接回答 + 引用政策条款
- 求推荐：推荐商品 + 推荐理由
- 转人工："我帮你转接人工客服"
```

### 第三步：配置插件

开启插件：
- **必应搜索**：查物流信息
- **链接读取**：查商品详情页

### 第四步：设置变量

| 变量 | 作用 | 示例值 |
|------|------|--------|
| `order_lookup` | 存储查到的订单 | 点击"查订单"时触发 |
| `transfer_reason` | 记录转人工原因 | "无法识别问题" |

### 第五步：发布

发布到网页和微信渠道。

---

## 方案二：用 Dify 实现（2 小时）

### 第一步：部署 Dify

```docker compose -f docker-compose.yml up -d
```

### 第二步：配置模型

接入 GPT-4o-mini（性价比最高）。

### 第三步：搭建工作流

```
[用户输入]
    ↓
[意图识别节点 (LLM)]
    ↓
查订单 ──→ [调用订单 API (代码节点)]
    ├── 成功 → [生成回复 (LLM)]
    └── 失败 → [转人工]
    ↓
问政策 ──→ [检索知识库]
    ↓       [生成回复 (LLM)]
    ↓
求推荐 ──→ [检索商品库]
    ↓       [调用推荐算法 (代码节点)]
    ↓       [生成推荐 (LLM)]
    ↓
其他 ──→ [转人工节点]
```

### 第四步：实现订单查询

创建代码节点调用订单 API：

```python
import requests

def main(data: dict) -> dict:
    order_id = data.get("order_id", "")
    # 调用订单系统 API
    resp = requests.get(
        f"https://api.example.com/orders/{order_id}",
        headers={"Authorization": "Bearer YOUR_TOKEN"}
    )
    if resp.status_code == 200:
        return resp.json()
    return {"error": "查不到该订单"}
```

### 第五步：日志监控

用 Dify 的日志功能跟踪：
- 每天有多少咨询？
- 最常见的问题是什么？
- 转人工率有多高？
- 平均响应时间？

## 测试清单

上线前要测试这些场景：

| 场景 | 测试问题 | 预期结果 |
|------|----------|----------|
| 常见问题 | "退货要多久？" | 引用知识库回答 |
| 订单查询 | "我的订单 ORD-001 到哪了？" | 返回订单状态 |
| 模糊查询 | "我想买个好用的耳机" | 推荐商品并问预算 |
| 无法处理 | "帮我写首诗" | 转人工或友好拒绝 |
| 多轮对话 | 先问价格再问发货 | 记住上下文 |

---

## 上线后的优化

### 分析日志

每周看一次日志，回答这些：
- 用户最常问哪 3 类问题？
- 有哪几个问题是 Bot 回答错的？
- 转人工的主要原因是什么？

### 持续改进

1. **补充知识库**：把 Bot 答错的问题加到知识库
2. **优化 Prompt**：针对高频问题优化回复逻辑
3. **添加新功能**：根据用户需求迭代
4. **A/B 测试**：对比不同 Prompt 的效果

---
