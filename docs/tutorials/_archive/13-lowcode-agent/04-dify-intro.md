# 04 - Dify 入门：搭建本地 AI 应用

> 自己部署一个开源 AI 平台，数据完全掌控在自己手里

---

## 为什么选 Dify？

| 对比项 | Coze（云端） | Dify（自部署） |
|--------|-------------|---------------|
| 数据 | 存在字节跳动服务器 | 存在你自己服务器 |
| 费用 | 按量付费/订阅 | 只有服务器成本 |
| 定制 | 有限的定制 | 完全开源可改 |
| 模型 | 只支持 Coze 集成的 | 可接入任意模型 |
| 私有化 | ❌ | ✅ 完全私有 |

---

## 部署方式

### 方式一：Docker 部署（推荐）

**前提**：需要安装 Docker 和 Docker Compose

```bash
# 1. 下载 Dify
git clone https://github.com/langgenius/dify.git
cd dify/docker

# 2. 复制配置
cp .env.example .env

# 3. 启动
docker compose up -d
```

启动后访问 `http://localhost:3000` 就能看到 Dify 界面。

### 方式二：云服务

如果不想自己部署，可以使用 Dify Cloud：
- **dify.ai** 官方托管版
- 直接注册即可使用

## 基础配置

### 1. 添加模型

首次进入 Dify 后，需要配置 LLM 模型：

1. 进入 **设置 → 模型供应商**
2. 选择你用的模型（OpenAI、Claude、通义千问等）
3. 输入 API Key
4. 保存

**推荐**：如果只是学习，用 OpenAI 的 GPT-3.5-turbo 最划算。

### 2. 创建应用

1. 点击 **"创建应用"**
2. 选择 **"对话型应用"**
3. 输入名称，比如"学习助手"
4. 进入编排页面

---

## 应用编排

Dify 的编排页面分三块：

### 左边：提示词编排

类似 Coze 的 Prompt 编辑：

```
你是一个学习助手，擅长用通俗的语言解释复杂概念。
当用户提问时，按照以下步骤：
1. 用一句话概括
2. 拆解核心概念
3. 举个例子
4. 检查用户是否理解
```

### 右边：预览调试

输入测试问题，实时看回复效果。

### 上下文：对话变量

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `{{sys.query}}` | 用户当前问题 | "什么是 Agent？" |
| `{{sys.dialog}}` | 对话历史 | [用户消息, AI 回复] |
| `{{sys.dataset}}` | 知识库检索结果 | [相关段落] |

## 添加知识库

1. 进入 **"知识库"** 页面
2. 点击 **"创建知识库"**
3. 上传文档（支持 PDF、TXT、Markdown、网页）
4. 选择分段策略
5. 系统自动处理向量化
6. 在应用设置中关联这个知识库

### 分段策略对比

| 策略 | 说明 | 适合场景 |
|------|------|----------|
| 自动分段 | Dify 自动切分 | 通用文档 |
| 自定义分段 | 设定大小和重叠 | 技术文档 |
| Q&A 分段 | 一问一答为一段 | FAQ、手册 |

## 发布应用

Dify 支持多种发布方式：

| 方式 | 说明 | 操作 |
|------|------|------|
| 网页嵌入 | 在网站中嵌入聊天窗口 | 复制 iframe 代码 |
| API | 通过接口调用 | 生成 API Key |
| 分享链接 | 分享给他人使用 | 开启公开访问 |
