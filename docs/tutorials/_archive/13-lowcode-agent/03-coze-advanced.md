## 插件深入

### 插件 vs 工具

在 Coze 里，插件 = 一系列工具的集合。

例如"必应搜索"插件包含：
- `web_search`：网页搜索
- `news_search`：新闻搜索
- `image_search`：图片搜索

### 热门插件推荐

| 插件 | 功能 | 使用场景 |
|------|------|----------|
| 必应搜索 | 联网搜索 | 实时信息查询 |
| AI 绘画 | 文生图 | 生成配图、设计素材 |
| 链接读取 | 抓取网页内容 | 总结文章、分析页面 |
| PDF 解析 | 读取 PDF | 文档问答 |
| 代码解释器 | 执行 Python | 数据分析、图表生成 |

### 自定义插件

Coze 支持创建你自己的插件（需要简单的 API 知识）：

1. 进入 **插件市场 → 创建插件**
2. 输入 API 信息（URL、参数、认证方式）
3. 定义好输入输出格式
4. 保存后 Bot 就能调用

## 知识库深入

### 知识库格式支持

| 格式 | 说明 | 限制 |
|------|------|------|
| 文本 | .txt 文件 | 每个文件 < 10MB |
| PDF | 扫描件/电子版 | 支持 OCR 识别 |
| 网页 | 输入 URL 自动抓取 | 单次最多 100 页 |
| 表格 | CSV/Excel | 自动解析为结构化数据 |
| 飞书文档 | 飞书在线文档 | 需绑定飞书账号 |

### 分段策略

Coze 会自动把长文档切成小段（chunk），Bot 只检索相关的段落。

**分段参数**：
- **分段大小**：默认 500 字，越小越精准但上下文少
- **重叠**：段与段之间的重叠字数，保证信息不丢失

**调优建议**：
- 技术文档 → 分段大一点（800-1000 字），保持上下文完整
- FAQ 问答 → 分段小一点（200-300 字），精准匹配

## 工作流（Workflow）

工作流是 Coze 最强大的功能——把多个步骤串起来，让 Bot 按你设定的流程执行。

### 可视化工作流

工作流编辑界面像流程图：

```
[开始] → [输入处理] → [调用插件] → [判断条件]
                                           ↓
                                    [是] → [调用 API]
                                           ↓
                                    [否] → [直接回复]
                                           ↓
                                    [输出处理] → [结束]
```

### 工作流节点类型

| 节点 | 功能 | 示例 |
|------|------|------|
| LLM | 调用大模型 | 让 GPT-4 分析文本 |
| 插件 | 调用插件 | 搜索、画图、读网页 |
| 代码 | 执行代码片段 | 处理数据格式 |
| 条件 | 分支判断 | 如果/否则 |
| 循环 | 重复执行 | 遍历列表 |
| 变量 | 存储中间结果 | 保存临时数据 |
| 知识库 | 检索文档 | 查产品手册 |

### 实战：自动客服工作流

```
1. 用户输入问题
2. LLM 判断意图（售前/售后/其他）
3. 如果售前 → 查产品知识库 → LLM 生成回复
4. 如果售后 → 查工单系统 API → 返回工单状态
5. 如果其他 → 转人工
```

## 变量和记忆

### 变量类型

- **会话变量**：当前对话中有效，对话结束清空
- **用户变量**：跨会话保存，记住用户偏好
- **全局变量**：所有用户共享，适合配置信息

### 实战：记住用户偏好

```
用户：我喜欢简洁的回答
Bot：（保存用户变量 preference=简洁）
下次对话：
用户：推荐几本编程书
Bot：（读取到用户偏好简洁，给出一句话推荐）
```

## 优化 Bot 的技巧

1. **Prompt 要结构化**：用"如果…就…"的句式，而不是一堆描述
2. **少用否定**：说"要做什么"而不是"不要做什么"
3. **加示例**：给 Bot 2-3 个问答例子，它会学得更快
4. **用变量减少重复**：常见信息存成变量，不要在 Prompt 里硬编码
5. **测试再测试**：用不同的输入反复测试，找到边界
