## 🎯 核心概念

### 为什么需要 Sandbox？

1. **安全性** - 隔离不可信代码，防止恶意操作
2. **资源控制** - 限制 CPU、内存、网络、文件系统访问
3. **故障隔离** - 单个 Agent 崩溃不影响主系统
4. **权限最小化** - 按需授予最小必要权限
5. **可审计性** - 记录所有沙箱内操作

### Sandbox 类型对比

| 类型 | 隔离级别 | 性能开销 | 适用场景 |
|------|----------|----------|----------|
| VM 虚拟机 | ⭐⭐⭐⭐⭐ | 高 | 完全隔离、多租户 |
| Docker 容器 | ⭐⭐⭐⭐ | 中 | 微服务、工具执行 |
| Node.js vm 模块 | ⭐⭐⭐ | 低 | JS 代码安全执行 |
| Worker Threads | ⭐⭐ | 低 | 计算密集型任务 |
| iframe | ⭐⭐⭐ | 低 | 浏览器端 UI 隔离 |
| Deno 权限模型 | ⭐⭐⭐⭐ | 低 | Deno 运行时安全 |
